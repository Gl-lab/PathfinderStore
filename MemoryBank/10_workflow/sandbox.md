# Sandbox Workflow

## Когда читать

Читать перед запуском команд, которые пишут в `.git`, `bin/`, `obj/`, создают миграции или запускают `dotnet build/test/ef`.

## Главное правило

Если команда выглядит корректной, но падает с access/lock/build symptom из этого файла, сначала подозревать ограничение среды, а не ошибку кода.

В текущей Codex-среде approval policy может не позволять escalation. В таком случае нужно:

- зафиксировать симптом;
- не чинить lock/temp-файлы вручную без необходимости;
- не делать вывод о поломке кода по одному sandbox-сбою;
- при возможности повторить команду позже вне ограниченной среды.

## Подтверждённые симптомы

### Git

`git add` и другие операции, которые должны обновить индекс, могут падать с ошибкой:

```text
fatal: Unable to create '.git/index.lock': Permission denied
```

Если появилась ошибка про `.git/index.lock` внутри sandbox, не считать это проблемой git-репозитория и не тратить время на ручную чистку lock-файла.

### Dotnet

`dotnet build` внутри sandbox может завершаться сообщением вида:

```text
Ошибка сборки.
Ошибок: 0
```

`dotnet test` внутри sandbox может завершаться с `Exit code: 1` и почти пустым выводом, хотя вне sandbox тесты проходят.

Ошибки доступа к временным файлам в `obj/` чаще всего означают ограничение sandbox, а не проблему в коде.

## Что делать

Если операция относится к одному из сценариев ниже, после первого же подозрительного сбоя не тратить много времени на обходы:

- `git add`, `git commit` и другие операции, которым нужно писать в `.git`;
- `dotnet build`, `dotnet test`, `dotnet ef`, если вывод неполный или противоречивый;
- любые команды, которые пишут во временные файлы внутри `obj/`, `bin/` или создают lock-файлы.

## Чего не делать

- Не пытаться чинить `.git/index.lock` удалением файлов, если ошибка возникла только внутри sandbox.
- Не делать вывод о сломанной сборке по одному сообщению `Ошибка сборки.` без нормальной диагностики.
- Не тратить много времени на повторные запуски одной и той же команды, если уже видно характерный symptom ограничения среды.
